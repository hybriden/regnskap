# Regnskapssystem

Norsk regnskapssystem med full støtte for dobbelt bokholderi, SAF-T, MVA, EHF-fakturering, og norsk regnskapslov.

## Tech Stack

- **Backend:** ASP.NET Core 10 + EF Core 10 + PostgreSQL 16
- **Frontend:** React 19 + TypeScript + Vite
- **Architecture:** Modular monolith med feature folders
- **External formats:** SAF-T XML, EHF/PEPPOL, CAMT.053, pain.001

### Architecture
- **Pattern:** Pipeline + Fan-out + Producer-Reviewer (composite)
- **Execution mode:** Subagents

### Usage
- Full pipeline: `/regnskap:run {task description}`
- Single module: `/regnskap:modul kontoplan`
- Compliance audit: `/regnskap:revisjon`
- To update the harness: `shipwright:harness`

### Module Build Order

```
1. Kontoplan          (ingen avhengigheter)
2. Hovedbok           (avhenger av: kontoplan)
3. Bilagsregistrering (avhenger av: hovedbok)
4. MVA-håndtering     (avhenger av: kontoplan, bilag)
5. Leverandørreskontro(avhenger av: bilag, hovedbok)
6. Kundereskontro     (avhenger av: bilag, hovedbok)
7. Fakturering        (avhenger av: kundereskontro, MVA)
8. Bankavstemming     (avhenger av: hovedbok, leverandør, kunde)
9. Rapportering       (avhenger av: alle moduler)
10. Periodeavslutning (avhenger av: hovedbok, rapportering)
```

## Conventions

- **Språk i kode:** Norske domenenavn (Bilag, Konto, Postering), engelske tekniske termer (Service, Controller, Repository)
- **Beløp:** Alltid `decimal`, aldri `float`/`double`. Value object `Belop` der mulig.
- **Sletting:** Alltid soft delete (IsDeleted). Aldri hard delete.
- **Revisjonsspor:** Alle entiteter arver `AuditableEntity` med CreatedAt/CreatedBy/ModifiedAt/ModifiedBy.
- **Dobbelt bokholderi:** Valideres i domain-laget. Debet = kredit er en hard invariant.
- **MVA:** Grunnlag og MVA-beløp er alltid separate felt. Aldri beregn fra sum.
- **Tester:** Hver forretningsregel skal ha minst én unit test.
